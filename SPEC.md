1. 架构哲学：奥卡姆剃刀
核心目标: 打造跨平台、零侵入、配置与运行彻底解耦的极简 Xray 代理引擎。

设计准则: "前台运行，随终端生灭"。摒弃后台守护逻辑，不污染系统代理环境。将"订阅转换"与"代理运行"剥离，仅保留最纯粹的启动与托管骨架。

2. 空间拓扑与交付约束

四大核心组件:

- Xray Core: 核心二进制驱动。
- converter.sh (订阅转换器): 将 vless:// 链接解析为自完备的 Xray JSON 配置（内含路由免疫规则），存入 profiles/ 仓库，同时自动激活至 run/ 沙箱。
- shield.sh (启动脚本): 仅负责环境校验与前台进程托管。支持传参快速切换节点。
- nodes.sh (节点管理控制台): 交互式终端菜单，展示节点列表与 TCP 连通状态，支持编号切换与删除。

物理隔离与运行专区:

- 配置仓库 (profiles/): 专门存放所有由转换器生成的节点配置文件，多文件优雅共存，管理一目了然。
- 运行专区 (run/): 仅包含当前正在生效的唯一配置 config.json 和身份标签 node.name。绝对纯净，极度收敛 Xray 的运行时读取范围。

端口避让原则:

摒弃高频碰撞的 10808/10809。改用非常规的高位隐蔽端口（SOCKS5 使用 37080，HTTP 使用 37081），最大程度避免与其他开发工具或代理软件的冲突。

优雅的节点切换:

1. 转换器自动激活：每次生成新配置时，自动清洗 run/ 并复制 config.json + node.name 激活为当前状态。
2. 命令行直切：./shield.sh <节点名.json> 自动清空 run/ 并拷入指定配置后启动。
3. 交互式管理：./nodes.sh 展示带 TCP 测速的节点列表，输入编号即可切换或删除。

3. 线性生命周期 (启动脚本极简逻辑)

Agent 必须严格按照以下线性流程编写，保持逻辑的绝对干净：

步骤 1：开局物理斩断 (Fail-Fast)

- 无核心则死: 探测 xray 二进制是否存在且可执行。
- 空目录则死: 探查 profiles/ 文件夹是否存在。

步骤 2：优雅寻址与装载

- 若传入参数，从 profiles/ 取出指定节点，清洗 run/ 并部署。
- 否则直接读取 run/config.json 作为当前配置。
- 从 run/node.name 读取节点身份用于 banner 显示。
- 提取配置内的端口号用于后续校验。

步骤 3：接管与生灭

- 端口校验: 探测 SOCKS5/HTTP 端口是否被占用，冲突则报错退出。
- 极简通告: 显示节点名、监听地址、退出方式。
- 阻塞托管: exec xray -c run/config.json，以当前进程替换 shell。
- 随窗生灭: 终端关闭即进程销毁，无需任何清理逻辑。
