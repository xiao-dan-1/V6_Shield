1. 架构哲学：奥卡姆剃刀
核心目标: 在任意桌面操作系统上实现零侵入、免安装的极简 Xray 代理客户端。

设计准则: “前台运行，随终端生灭”。坚决不写后台守护逻辑，不污染系统网络设置（不改系统代理）。剔除一切非必要的 UI 交互和防御代码，仅保留维持核心运转的绝对骨架。

2. 交付物与执行环境约束
极简交付: 仅允许存在两个核心组件：Xray 核心程序（针对当前架构的二进制文件）和 统一启动脚本。

环境抗性红线:

执行路径锁死: 启动脚本必须使用绝对路径或可靠的相对路径定位核心程序，避免工作目录漂移导致的路径死锁。

编码死锁: 必须确保终端环境输出为 UTF-8。生成的临时配置必须为 .temp_config_$PID.json，且强制使用无 BOM 的 UTF-8 编码。

绝对禁令: 零日志文件夹（不落盘日志）、零图形界面、零用户输入提示框。

3. 线性生命周期 (启动脚本极简逻辑)
Agent 必须严格按照以下线性流程编写脚本，禁止任何复杂的系统级分支循环或交互式等待：

步骤 1：开局双重斩断 (Fail-Fast)

无核心则死: 检测当前同级目录有无 Xray 核心二进制文件。无则终端红字报错，等待用户确认后退出。

无链接则死: 直接读取系统剪贴板。若不包含 vless:// 前缀，直接红字报错 "剪贴板未检测到 VLESS 链接，请复制后重试"，等待用户确认后退出。（严禁提供手动输入框或文件读取逻辑）

步骤 2：解构与渲染

硬核解析: 提取剪贴板链接中的 address, port, uuid, sni, pbk, sid, fp, flow。必须使用 URL Decode 解码参数。

极简注入: 将参数注入脚本内置的极简 JSON 模板（仅保留 inbound 10808 SOCKS / 10809 HTTP 和 outbound Reality，剔除一切冗余 routing 规则）。落盘为 .temp_config_$PID.json。

步骤 3：接管与生灭

端口校验: 探测 10808/10809 端口，若被占用则红字报错退出。

通告与运行: 屏幕打印极简绿字提示：

Plaintext
>>> Xray 代理已前台启动 | SOCKS5: 10808 | HTTP: 10809
>>> 节点: {Address}
>>> 停止代理请直接关闭本终端窗口 [X]
阻塞托管: 前台直接执行 ./xray -c .temp_config_$PID.json（或等效的核心调用命令），将标准输出流直接交给当前终端接管。

死后清理: 在执行命令的下一行（或利用系统级的 Exit Trap/Hook），写入删除 .temp_config_$PID.json 的代码。只有当终端未被强杀，或者 Xray 核心自身主动崩溃退出时，才会执行到此步，作为生命周期的物理兜底。
