1. 架构哲学：奥卡姆剃刀
核心目标: 打造跨平台、零侵入、配置与运行彻底解耦的极简 Xray 代理引擎。

设计准则: “前台运行，随终端生灭”。摒弃后台守护逻辑，不污染系统代理环境。将“订阅转换”与“代理运行”剥离，仅保留最纯粹的启动与托管骨架。

2. 空间拓扑与交付约束
三大核心组件:

Xray Core: 核心二进制驱动。

Converter Tool (订阅转换器): 独立的小型工具，专职负责将 vless:// 等订阅链接解析并转换为标准 JSON 配置。

Boot Script (统一启动脚本): 仅负责环境校验与进程托管。

物理隔离与运行专区:

配置仓库 (profiles/): 专门存放所有由转换器生成的节点配置文件，多文件优雅共存，管理一目了然。
运行专区 (run/): 仅包含当前正在生效的唯一配置 config.json。绝对纯净，不能有任何其他文件。极度收敛 Xray 的运行时环境读取。

端口避让原则:

摒弃高频碰撞的 10808/10809。改用非常规的高位隐蔽端口（例：SOCKS5 使用 37080，HTTP 使用 37081），最大程度避免与其他开发工具或代理软件的冲突。

优雅的节点切换:

1. 转换器使用：每次生成新配置时，自动将其复制更新到 run/config.json 激活其为当前状态。
2. 命令行直切：用户可通过 ./shield.sh <节点名.json>，脚本将自动清空 run/ 目录以保证绝对纯粹，并拷入指定配置为 config.json 然后启动。零困惑，极度优雅。

3. 线性生命周期 (启动脚本极简逻辑)
Agent 必须严格按照以下线性流程编写，保持逻辑的绝对干净：

步骤 1：开局物理斩断 (Fail-Fast)

无核心则死: 探测核心程序是否存在。无则红字静默退出。

空目录则死: 探查 profiles/ 文件夹。若未发现任何有效的 .json 配置文件，终端红字输出提示：“未检测到配置文件，请先使用转换工具导入 VLESS 节点”，随后退出。

步骤 2：优雅寻址与装载

配置接管: 自动读取 profiles/ 目录下指定的配置文件（可默认加载最新生成的文件，或按字母顺序的首个文件）。

提取该配置内部设定的高位端口（如 37080）用于后续校验。

步骤 3：接管与生灭

端口校验: 探测目标隐蔽端口（如 37080/37081）是否被占用，若遇冲突则报错退出，保障零侧漏。

极简通告: 屏幕打印清爽绿字提示：

Plaintext
>>> Xray 引擎已挂载 | 状态: 活跃
>>> SOCKS5: 37080 | HTTP: 37081
>>> 配置来源: profiles/current_node.json
>>> 终止代理请直接关闭本终端窗口 [X]
阻塞托管: 前台执行 ./xray -c profiles/current_node.json，将标准输出流直接交付给当前终端。

随窗生灭: 进程与终端窗口绑定，窗口关闭即进程销毁。由于配置已固化在 profiles/ 目录中，无需执行任何死后文件清理逻辑，实现极致优雅的生命周期闭环。
