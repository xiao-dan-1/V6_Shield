## 2026-04-23 迭代一：基础双端引擎与动静隔离

### 1. 需求分析与架构敲定
本项目的核心目标是抛弃臃肿的第三方运行库与 GUI 客户端，为纯粹的 Linux 终端环境打造一个无感化代理护盾。
项目被物理拆分为三个极简核心结构：
- **`converter.sh`**（订阅转换器）：负责将节点链接转化为配置。完全以 Bash 编写，首发版本即实现了对 `WS+TLS`, `TCP+Reality`, `gRPC+TLS` 流派的全面兼容。
- **`shield.sh`**（启动枢纽）：代理生命管理脚本，直接利用 `exec` 实现前台接管。启动流极为线性可控：`Fail-Fast 预检 → 参数装载 → 端口预检 → 宿主托管`。
- **`profiles/`**（纯净仓库）：负责存放生成的 JSON 节点图纸文件，建立基础的物理隔离。

### 2. 成果验证与纠偏
MVP 阶段通过终端 `curl` 跑通了连接，明确了一项开发原则中的细节——为防止 DNS 泄漏，SOCKS5 调用必须强制使用 `socks5h://` 以指向远端解析，同时兼容了 37080(SOCKS) / 37081(HTTP) 两路流量分发。

## 2026-04-23 迭代二：显式状态机与绝对纯净沙箱

### 1. 隐式隐患与 Unix 哲学介入
早期的 `shield.sh` 使用了“根据时间戳扫描最新文件”来判定激活节点的隐式逻辑，此举既不透明也缺乏掌控力。
由于秉持正确且优雅的前提，项目引入经典 Unix 哲学的“符号链接 (Symlink)”：在 `profiles/` 下建立明确的 `active.json` 指针。现在用户输入 `ls -l profiles/` 即可直接从原生系统反馈中一目了然看清挂载锚点。

### 2. 沙箱隔离级别的巅峰重构
指针概念随后被进一步升华为绝对的**运行环境与配置文件分离**架构。
设立了绝对高敏的沙箱执行区 **`run/`**。
不论军火库 `profiles/` 内存留多少订阅配置，`shield.sh` 的逻辑被彻底梳理为：`清除并格式化 run/ 跑道 → 将待定 JSON 注入 run/config.json → 起飞`。所有多余配置文件不参与真实运行周期，消除了配置误读风险。

## 2026-04-23 迭代三：交互中枢 `nodes.sh` 与内建测速探针

### 1. 技术痛点与方案抉择
缺失节点直观反馈意味着系统仍有残缺。但如果不借助完整的 Xray 真连接或是外部工具，如何在不违背“极简”前提下测试百余节点的响应质量？
我们采用了最底层的内置魔法——基于 **`/dev/tcp`** 的原生 Bash 连接模块。

### 2. 核心功能实现与热修复
创建了纯净数字菜单驱动的 `nodes.sh`。具备动态标识 `[活跃]`、巡检与热销毁 (`d`) 能力。
- **TCP 闪电测速的接入**：在交互前，发起一秒微小超时的毫秒级 Socket 返回并宣读。
- **提取逻辑 Bug 阻击**：开发中暴露了一个盲区，提取程序误将配置文件第一部分的本地拦截端（37080）作为目标，导致无差别的“超时/阻断”。紧急下发正则表达式热修复，精准切出 `"vnext"` 段并抓取真实靶向目标，修复了延迟读数。
- 对比 v2rayN 等工具的全管线真测速，我们主动克制并接纳了这种局部的“误差”，将理念明确定位为**“判定能否存活与可用”的探针**，而非牺牲系统资源去复现真实的跨海握手时长。

## 2026-04-23 迭代四：双模系统级代理注入 (`proxy.sh`)

### 1. 环境代理脱节问题
Xray 的基础分发已经趋于稳健，但图形化工具或终端并不会自动承接 HTTP/Socks5 出口，必须引入单独的系统代理开关。且不能接受安装外部守护进程的传统方案。

### 2. 极致的幂等渗透接管
从分析 v2rayN 的重度设定脚本入手，利用极简思路编写出具备自动化感知能力的双模脚本 `proxy.sh`：
- **桌面级 (Desktop)**：调用底层接口拨动 `gsettings` 的网络框架参数。
- **终端级 (CLI RC 注入)**：创造性引出了一劳永逸的“**Fenced Block (标记围栏)**”概念。
  利用正则表达式向特定环境配置文件 (`.bashrc` / `.zshrc`) 注入含有开启/闭合定界符的网络出口环境变量。它不依赖 `source` 命令跨实例执行，亦不污染原有 RC 逻辑，真正做到了按一下回车，即在系统生态中达到绝对的持久化挂载与平滑擦除。

## 2026-04-24 迭代五：奥卡姆剃刀与法则内联 (Inline Routing)

### 1. 独立路由文件的悖论
系统曾采用读取外部 `routing.json` 的形式来防止“隧道自噬”和剥离私网流量。但在今日极简哲学的再次审视下，发现这些防御性路由规则对于所有代理节点来说，本质是绝对恒定的“网络物理定律”，将其当做外部挂靠文件极不优雅，也打破了沙箱文件单一性的承诺。

### 2. 内联化深度重构
果断挥动奥卡姆剃刀，将路由规则彻底**烘焙（Baking）**进了 `converter.sh` 生成链的底座中：
- 将包含本地回环 (`127.0.0.1`, `::1`) 以及节点真实服务器 IP 黑名单的 `AsIs` 直连规则强加密封入每一次 `config.json` 的产出过程中。
- **架构收益**：彻底全网剔除孤立的 `routing.json`。`shield.sh` 减少了 15% 的周边校验代码，沙箱 `run/` 首次具备了绝对纯粹的 100% 自闭环与单体自容错能力。
